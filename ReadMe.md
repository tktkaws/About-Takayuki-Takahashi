# Job-Hunting-Record

就職活動の管理をGitHubFlowに沿って行うという試み

1. 1週間の活動目標issueをたてる
2. issueのブランチ(1week)を切る
3. ブランチ(1day)を切る
4. 日毎にPRを出し自身でレビュー・merge
5. 1週間毎にPRを出し自身でレビュー・merge

###### 【メリット】
- 就職活動を俯瞰的に見て整理できる
###### 【就活の基本戦略】

受託開発をメインに、シェアリングエコノミー・ヘルスケア関連であれば自社開発にも応募する。

snsで評判の良いSESがあればそこも受ける。
