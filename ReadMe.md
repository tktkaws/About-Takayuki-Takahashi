# Job-Hunting

就職活動の管理をGitHubFlowに沿って行うという試み

就職活動を俯瞰的に見て整理するために、
Railsプロジェクトのようなディレクトリ構成に当てはめる

1. issueをたてる
2. ブランチを切る
3. 一定の成果が出たらPR
4. 別視点でその成果をreview
5. 問題なければMasterにmerge


###### 【メリット】
 - Markdown記法に更に慣れる
 - 自分が今何をやるべきか整理できる、はず
