# Job-Hunting-Record

就職活動の管理をGitHubFlowに沿って行うという試み

1. issueをたてる
2. ブランチを切る
3. 一定の成果が出たらPR
4. 別視点でその成果をreview
5. 問題なければMasterにmerge

###### 【メリット】
- 就職活動を俯瞰的に見て整理できる
- Markdown記法に更に慣れる
 
 
###### 【基本戦略】

受託開発をメインに、シェアリングエコノミー・ヘルスケア関連であれば自社開発にも応募する。
snsで評判の良いSESがあればそこも受ける。

- ポートフォリオの質を可能な限り高める
- wantedly以外からエントリーする
- Railsには拘らない(laravelでも良い)
- 自己PRの切り口を変える
- このJob-Hunting-Recordも差別化の手段
- vue.jsのポートフォリオも作成する
